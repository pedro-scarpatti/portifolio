Projetos Principais

⭐ projeto principal

Simulador de Pipeline de Processamento de Dados

Problema: Pipelines de dados falham silenciosamente em produção — sem visibilidade de estado, duplicidade não detectada e logs inexistentes.

Como funciona: Interface que simula arquitetura real de fila + processor + idempotência. Jobs passam por estados (PENDENTE → PROCESSANDO → CONCLUÍDO/ERRO), deduplicação via hash, log estruturado em tempo real e simulação de falhas configurável.

JavaScript ES6+ FIFO Queue Idempotência (Hash) Event Bus Structured Logging Retry Pattern ETL Pattern Async/Await
Fila FIFO com prioridade e controle de concorrência (max 3 workers simultâneos)
Hash de idempotência detecta e bloqueia jobs duplicados antes do processamento
Estados PENDENTE → PROCESSANDO → CONCLUÍDO / ERRO com timestamps precisos
Log estruturado multi-nível: DEBUG, INFO, SUCCESS, WARNING, ERROR — exportável em JSON
Simulação de falhas configurável (0–50%) com erros realistas de produção
Módulos separados: services / processors / validators — mentalidade de engenharia real
